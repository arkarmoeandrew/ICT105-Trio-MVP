# Lab 04 – MVP Feature List

## MVP Decision Rule
The MVP should include the smallest set of features that can demonstrate the core problem–solution fit by the end of the semester.

## MoSCoW Prioritization

| Feature ID | Feature Name | Problem Solved | Related User Story | Priority (Must/Should/Could/Won't) | Technical Complexity (1–5) | User Value (1–5) | Evidence Strength (1–5) | Include in Final Prototype? |
|------------|--------------|----------------|--------------------|------------------------------------|----------------------------|------------------|-------------------------|------------------------------|
| F01 | RSU Student Login | Ensures only verified students access the platform | US-01 | Must | 2 | 5 | 5 | Yes |
| F02 | Create Resource Listing | Allows students to post equipment, learning resources, or services | US-02 | Must | 3 | 5 | 5 | Yes |
| F03 | Search & Filter Marketplace | Helps students quickly find needed resources or services | US-03 | Must | 3 | 5 | 5 | Yes |
| F04 | View Listing Details | Lets students view complete information before contacting the owner | US-04 | Should | 2 | 4 | 4 | Yes |
| F05 | Contact Resource Owner | Allows students to request a resource or service | US-05 | Must | 3 | 5 | 4 | Yes |
| F06 | Manage My Listings | Allows users to edit or delete their own listings | US-06 | Could | 3 | 4 | 3 | Optional |
| F07 | Resource Booking / Reservation | Reserve resources before collecting them | Future | Won't | 5 | 4 | 2 | No |

## Must-Have Features
- **RSU Student Login (F01)** – Verify users using RSU email authentication.
- **Create Resource Listing (F02)** – Students can create listings for equipment, learning resources, or services.
- **Search & Filter Marketplace (F03)** – Search by keyword and category.
- **Contact Resource Owner (F05)** – Contact or request resources/services from the listing owner.

## Should-Have Features
- **View Listing Details (F04)** – Display complete resource information before contacting the owner.

## Could-Have / Future Features
- **Manage My Listings (F06)** – Edit or delete personal listings.

## Not in MVP
**Resource Booking / Reservation (F07)** is intentionally excluded from the MVP because it has high technical complexity, weaker supporting evidence, and is not essential for demonstrating the core value of the platform. It can be added as a future enhancement after validating the MVP.
